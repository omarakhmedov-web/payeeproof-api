Security model

PayeeProof should be a control layer, not a new trust problem.

No custody

PayeeProof does not hold funds and does not execute transfers. It exists to verify or analyze, not to move money.

Fail-closed

If details are invalid, mismatched, or suspicious, the safe answer is block, reverify, or escalate — not “probably fine.”

Live chain checks

This version uses live RPC lookups for destination classification and transaction analysis instead of purely static form logic.

API shape

Two public-facing product surfaces. One backend.

What the backend now does

PreflightAddress validation, expected-vs-provided comparison, live on-chain destination classification.
RecoveryTransaction lookup, receipt/status analysis, destination-type checks, practical recovery guidance.
